# Six-Week Day 23 - Main Switches Alternate Supplies and Isolation Boundaries

## Purpose
Map every relevant source and operating state before making a claim about a main switch or isolation boundary.

## Core workflow
**S-O-U-R-C-E-S:** Scope the section → Observe every source → Understand operating states → Relate sources to switching points → Construct the boundary map → Examine evidence and dependencies → State the bounded conclusion.

## Evidence and claim model
- Evidence grades: **supplied**, **corroborated**, **derived**, **assumed**, **missing/conflicting**.
- Claim grades: **observation**, **provisional classification**, **supported paper conclusion**, **authorised technical determination**.
- Source-state boundary ledger: source → state → path → switching point → controlled/potentially energised/unresolved section → evidence grade → claim grade → dependency → reopening trigger.
- A device label or normal operating response does not prove every source, conductor or state is controlled.

## Practical application
Construct a source-state boundary ledger for a fictional installation, grade every evidence item and claim, identify dependencies, write a bounded conclusion, and rebuild the analysis after a source or work-boundary change.

## Assessment relevance
Tests source completeness, state analysis, path tracing, boundary reasoning, evidence calibration, changed-condition transfer and clear communication of unresolved energisation risk. The 12-point rubric and critical-error gates are original educational readiness tools, not an official RTO pass mark.
