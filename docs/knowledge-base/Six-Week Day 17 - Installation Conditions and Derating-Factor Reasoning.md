# Six-Week Day 17 - Installation Conditions and Derating-Factor Reasoning

## Purpose

Classify a route section by section, grade the evidence for each thermal condition, prove the applicability dependencies for any correction method, and return the bounded adjusted-capacity conclusion to the complete design chain.

## Core workflow

**C-O-N-D-I-T-I-O-N:** Capture route → Observe and grade evidence → Name influences → Determine the reference condition → Identify applicability dependencies → Test interactions → Integrate adjusted capacity → Observe consequences → Note the bounded conclusion.

## Evidence and claim control

### Evidence grades

- **E1 supplied:** present in current scenario material but not independently corroborated.
- **E2 corroborated:** supported by consistent current records or an authorised source and matching record.
- **E3 derived:** calculated or inferred from supported inputs using a stated method.
- **E4 assumed:** an explicitly conditional, unverified branch.
- **E5 missing or conflicting:** evidence that prevents a bounded conclusion.

### Claim grades

- **C1 description:** records what supplied material shows.
- **C2 conditional classification:** states what would apply if named dependencies are confirmed.
- **C3 supported paper reasoning:** reaches a bounded educational conclusion using current authorised data and supported scenario evidence.
- **C4 authorised verification:** requires an appropriately authorised person and approved procedures; automated content cannot award it.

## Route-condition ledger

For each material section, record the boundary, evidence and grade, installation method, thermal influences, proposed correction source and scope, applicability dependency, interaction check, claim grade and reopening trigger.

A correction factor is not evidence that its condition exists. Arithmetic is valid only inside a branch whose route classification, source scope, dependencies and interaction method are supported.

## Reopening triggers

Reopen the design when route boundaries, loading of neighbouring circuits, ambient conditions, insulation coverage, containment, external heat, manufacturer instructions, source currency or conflicting records change. Return the effect to [[Six-Week Day 16 - Design Current Device Rating and Conductor Capacity Relationship]] and then to [[Six-Week Day 18 - Voltage-Drop Concepts and Calculation Workflow]].

## Practical application

Map a fictional route section by section, grade its evidence, audit assessor-supplied training factors, state each applicability dependency and interaction rule, record rejected conditions, and reopen the design under changed-route scenarios.

## Assessment relevance

The learner must distinguish base and adjusted capacity, justify each route section and condition, prevent unsupported factor use and double correction, preserve traceability, and explain which upstream and downstream decisions reopen. Critical errors include treating assumptions as verified, combining factors without an authorised interaction method, ignoring a material section or proposing unsafe access to resolve uncertainty.
