# Six-Week Day 22 - Functional Switching Isolation and Emergency Switching Distinctions

## Purpose
Separate operational control, electrical isolation, emergency action and mechanical-maintenance switching by purpose, hazard, source set, boundary and evidence strength.

## Core workflow
**S-W-I-T-C-H:** State purpose → Work out hazard and people exposed → Identify every source and energy form → Trace the required boundary → Check each claimed function against evidence → Hold the conclusion to the evidence.

## Evidence and claim model
- Evidence grades: **supplied**, **corroborated**, **derived**, **assumed**, **missing/conflicting**.
- Claim grades: **observation**, **provisional classification**, **supported paper conclusion**, **authorised technical determination**.
- Switching-function ledger: purpose → hazard/person → sources/energy → boundary → claimed function → evidence grade → device/arrangement evidence → claim grade → reopening trigger.
- Evidence for one function does not automatically prove another function on the same device.

## Practical application
Classify six fictional needs, create a separate evidence row for each claimed function, correct appearance-based misconceptions, identify reopening triggers and complete a changed-source transfer task.

## Assessment relevance
Supports precise terminology, purpose-first classification, source and boundary reasoning, evidence control, changed-condition transfer and rejection of unsupported device claims. The 12-point rubric and critical-error gates are original educational readiness tools, not an official RTO threshold.
