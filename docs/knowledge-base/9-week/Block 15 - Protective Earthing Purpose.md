# Block 15 - Protective Earthing Purpose

## Core idea

Protective earthing is understood through purpose and fault-path reasoning, not slogans. Keep the normal load-current path separate from the protective path intended for fault conditions.

## Key distinctions

- A protective conductor is not an intended normal load-current return conductor.
- An exposed conductive part is considered in relation to a credible fault.
- A complete fault path must return to the source; drawing only the local earth connection is incomplete.
- Exact arrangements and disconnection conditions require authorised verification.

## Retrieval prompts

- What conductive part could become live under a fault?
- What is the normal current path?
- What abnormal connection creates the fault path?
- What evidence is still needed before claiming protective performance?

## Safety boundary

Earthing design, continuity testing, fault-loop assessment and disconnection decisions remain `reference_check_required`, `review-required` and subject to qualified review.
