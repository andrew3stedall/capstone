# Block 36 - Verification Evidence Model

## Core idea
Keep design intent, observed condition, test evidence, interpretation and conclusion distinct until a competent comparison against current authorised criteria supports a bounded conclusion.

## Retrieval prompts
- Why is documentation not proof of present condition?
- How should conflicting evidence be handled?
- What belongs in a traceable verification record?

## Safety boundary
No inspection sequence, test method, instrument, acceptance value or certification decision is supplied here. These require authorised sources, competent persons and qualified review.
