# Block 42 - Rest Reflection and Catch-Up

## Core idea
Use a fixed recovery boundary: diagnose closed-book, select one blocking gap, make one targeted correction, re-test with a changed example, record one carry-forward note and stop.

## Retrieval prompts
- What distinguishes a blocking gap from a non-blocking gap?
- Why use a changed-example re-test?
- What must remain unresolved rather than guessed?

## Safety boundary
This note repairs reasoning only. It does not resolve technical criteria, authorise field work or remove technical-review flags.
