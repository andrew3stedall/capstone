# Block 47 - Defect Communication Without Overclaiming

## Key idea
A useful defect message separates observed condition, possible consequence, uncertainty, formal classification and the decision being requested.

## Retrieval prompts
- Which terms usually require authorised-source support?
- How is a potential consequence different from a confirmed outcome?
- What makes an escalation request actionable?

## Application cue
Use the pattern: observed condition; bounded significance; unresolved evidence; requested authorised decision.

## Safety boundary
Do not invent defect categories, response times, compliance findings or remedies. Apply current authorised processes when actual risk is suspected.
