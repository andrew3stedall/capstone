# Block 61 - Error Analysis and Targeted Remediation

## Recall prompt
Can I distinguish an error symptom from the reusable weakness that caused it, then test one correction on a changed case?

## Remediation chain
1. Inventory errors
2. Separate symptom from root weakness
3. Classify repeated patterns
4. Rank by consequence and recurrence
5. Select one target
6. Build one correction artifact
7. Test transfer on a changed case
8. Record residual uncertainty

A successful retry on the same question is weaker evidence than transfer to a changed prompt.
