# Block 39 - Test-Order Reasoning

## Core idea
A defensible verification order follows dependencies, safety states, evidence preservation and explicit stop gates. It is provisional until checked against current authorised requirements and site conditions.

## Retrieval prompts
- What is a dependency between verification activities?
- Why can a state transition change the safety controls required?
- When should a stop gate prevent progression?

## Safety boundary
This note provides no field sequence, switching step, instrument setup or permission to test. Actual order and controls require authorised sources, approved procedures and competent persons.
