# Block 41 - Inspection-and-Test Integration Case

## Core idea
A verification case is an evidence chain, not a checklist. Scope, inspection, test purposes, sequence dependencies, expected evidence, contradictions, criteria and conclusion boundaries must remain traceable and distinct.

## Retrieval prompts
- What is an integration gap?
- Why can one favourable result not compensate for missing coverage?
- What belongs in a contradiction register?

## Safety boundary
This note does not authorise inspection, testing, switching, energisation, defect classification or certification. Actual work requires authorised documents, approved procedures and competent persons.
