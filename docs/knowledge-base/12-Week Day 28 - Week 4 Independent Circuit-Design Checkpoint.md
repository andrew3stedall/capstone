# 12-Week Day 28 - Week 4 Independent Circuit-Design Checkpoint

## Purpose

Tests whether the learner can coordinate Week 4 design reasoning under reduced support and prescribe evidence-based remediation.

## Core concepts

- **Checkpoint:** a bounded learning assessment, not an official assessment.
- **Critical error:** an error that overrides the numerical score.
- **Audit trail:** the chain from evidence through transformations to conclusion.
- **Remediation prescription:** one targeted task tied to observed evidence.

## C-H-E-C-K-P-O-I-N-T workflow

Clarify the boundary; harvest evidence; establish load cases; connect current, device and conductor roles; keep route sections distinct; preserve unresolved gates; observe changes; issue a bounded conclusion; note evidence grade; target remediation.

## Practical application

Learners complete a fictional independent design response, propagate changed conditions, apply a 20-point educational rubric and check critical-error gates.

## Assessment relevance

Observable evidence includes integration, source control, change propagation, conclusion discipline, self-assessment and remediation selection.
