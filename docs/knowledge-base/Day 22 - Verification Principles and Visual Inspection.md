# Day 22 - Verification Principles and Visual Inspection

## Core concept

Verification is a bounded judgement requiring a defined subject, known state, current criteria and sufficient converging evidence. Design, construction, visual, electrical-test and documentary evidence complement rather than automatically replace one another.

## Retrieval model

Use **V-E-R-I-F-Y**: Verify scope; Establish state; Retrieve criteria; Inspect before inference; Fit tests to purpose; Yield a bounded record.

Claims are graded as observed, supported, provisional or blocked. “Not demonstrated” means evidence is insufficient; it is not automatically a failure.

## Assessment evidence

The learner must build an evidence matrix, reconcile a contradiction, match test purpose to an unresolved question, reopen affected claims after a changed source state and issue a bounded conclusion. Invented methods, values or certification claims are critical errors.

## Safety and review boundary

No access, isolation, operation, testing, energisation or certification is authorised. Verification responsibilities, required records, inspection items, test purposes and sequence, acceptance criteria and certification duties remain `reference_check_required` and not `technically-reviewed`.
