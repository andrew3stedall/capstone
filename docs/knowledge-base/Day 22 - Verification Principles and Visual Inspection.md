## Core concept

Verification is a bounded judgement built from a defined subject, known installation state, current authorised criteria and sufficient converging evidence. Visual inspection is one evidence family alongside design, construction, electrical-test and documentary evidence; it cannot automatically replace them or be replaced by them.

The **V-E-R-I-F-Y** workflow is:

1. **Verify the scope**.
2. **Establish the state**.
3. **Retrieve the criteria**.
4. **Inspect before inference**.
5. **Fit tests to purpose**.
6. **Yield a bounded record**.

## Practical application

Use an evidence matrix for a fictional alteration to separate:

- what is visually observed;
- what documents establish;
- what an approved test is intended to determine;
- what remains inaccessible, conflicting or not demonstrated;
- which conclusion is supported and which must remain unresolved.

## Assessment relevance

This block prepares the learner to explain why verification is more than completing a checklist or obtaining favourable readings. It supports later work on mandatory tests, test sequence, result interpretation and fault finding while reinforcing safe scope control and evidence language.

## Safety and review boundary

This note does not provide an official checklist, test method, sequence, acceptance value, certification instruction or authority to access, isolate, energise, operate or test equipment. Current authorised sources and qualified technical review are required for all safety-critical procedures and final conclusions.
