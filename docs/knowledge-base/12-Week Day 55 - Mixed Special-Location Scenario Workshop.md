# 12-Week Day 55 - Mixed Special-Location Scenario Workshop

## Purpose

Integrate classification, source-state, design-evidence and inspection-evidence reasoning without collapsing distinct problems into one checklist.

## Core workflow

Use **L-A-Y-E-R-S**:

- **L** — Locate boundaries.
- **A** — Apply classification methods.
- **Y** — Yield a source map.
- **E** — Examine evidence by type.
- **R** — Relate dependencies and conflicts.
- **S** — State, stop and stress-test.

## Key distinctions

- Design intent is not proof of installed condition.
- A photograph is not proof of hidden construction.
- One site can contain several location and user boundaries.
- Alternate-source coverage must be evidenced rather than assumed.
- A material fact change reopens every dependent conclusion.

## Practical application

Create separate classification, source, design-evidence and inspection-evidence layers for an original mixed dossier. Then build a dependency map, identify conflicts, write bounded and unresolved claims, request missing evidence and revise after a staged change.

## Assessment relevance

The learner should demonstrate decomposition, applicability discipline, source-state awareness, evidence separation, dependency control and clear safety communication under an integrated scenario.

## Safety boundary

This note authorises no site classification, design approval, access, switching, isolation, testing, installation, alteration, energisation, commissioning, certification or verification.
