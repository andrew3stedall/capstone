# Six-Week Day 18 - Voltage-Drop Concepts and Calculation Workflow

## Purpose

Build an evidence-controlled voltage-drop calculation, grade the resulting claim correctly and return the result to the complete circuit-design chain.

## Core workflow

**V-O-L-T-A-G-E:** Verify boundary → Organise evidence → Locate authorised method → Translate units → Apply transparently → Gauge against criterion → Extend and reopen.

## Evidence and claim control

Use five evidence grades: E1 supplied, E2 corroborated, E3 derived, E4 authorised and E0 unresolved.

Use four claim grades: C1 description, C2 conditional calculation, C3 supported training conclusion and C4 qualified technical conclusion. This module supports educational C1–C3 work only.

The voltage-drop dependency ledger records the boundary, operating case, length basis, conductor data, method, criterion, result, dependencies and reopening triggers. Calculator precision does not upgrade weak evidence.

## Practical application

Prepare an evidence and dependency ledger, calculate only from fictional or authorised training data, identify the claim grade and mark every recalculation trigger. Reopen the result when the load, controls, source, boundary, route, conductor, arrangement, reference voltage, source edition or criterion changes.

## Assessment relevance

The learner must define the circuit boundary, use correct units, control the length convention, trace each material input to evidence, avoid remembered equations or limits, show arithmetic transparently and explain downstream reopening.

Critical errors include inventing inputs, mixing units or length conventions, treating remembered material as verified, presenting fictional arithmetic as approval or ignoring a changed dependency.
