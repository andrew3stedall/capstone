# Six-Week Day 32 - Motors Starting Conditions and Associated Protection Concepts

## Purpose

Build a traceable motor-system study conclusion by separating operating states, power and control paths, protection purposes, restart conditions and evidence dependencies.

## Core workflow

**M-O-T-O-R-S:** Map task and states → Obtain current applicable data → Trace power, control and energy paths → Outline protection purposes separately → Review grades, dependencies and changed conditions → State supported, unresolved, escalation and stop items.

## Evidence and claim model

- Evidence grades: **recalled**, **located**, **supported**, **transferred**, **unresolved**.
- Claim grades: **memory claim**, **provisional interpretation**, **supported study conclusion**, **authorised technical determination**.
- Motor-state evidence ledger: state or question → observed evidence → missing evidence → protection or control purpose → grades → dependencies → reopening trigger → bounded action.
- A nameplate current, catalogue statement, control label, protective-device rating or normal stop response is not a complete motor-system proof.

## Practical application

Complete motor-state evidence ledgers for fictional fan, pump and conveyor packs. Separate stopped, starting, running, overload, fault and supply-restored states; trace power and control paths; identify alternate sources and stored energy; grade evidence and claims; apply one changed condition; reopen dependent conclusions; and state bounded action without operating or testing equipment.

## Assessment relevance

Tests observable reasoning across state separation, terminology, path tracing, protection-purpose separation, evidence grading, dependency reopening, changed-condition transfer and bounded reporting. The 12-point rubric and critical-error gates are original educational tools, not an official RTO pass mark.

## Reopening triggers

Reopen the conclusion when the motor, driven load, duty, supply characteristics, source, starting method, conductor, protective device, control mode, automatic-restart behaviour, stored-energy condition, manufacturer information, jurisdiction, source currency or task boundary changes or conflicts with later evidence.

## References and review boundary

Exact motor data, starting performance, duty classifications, conductors, device coordination, settings, operating times, control arrangements, restart controls, isolation requirements and jurisdiction-specific duties remain `reference_check_required`.
