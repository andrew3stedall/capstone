# 12-Week Day 67 - Systematic Fault-Finding Workflow and Hypothesis Control

## Purpose

A scenario-based method for separating symptoms from observations, maintaining competing hypotheses and using discriminating evidence to revise conclusions without premature closure.

## Core model

Use **H-Y-P-O-T-H-E-S-I-S**: Hold symptom apart from fact; Yield a system boundary; Preserve observations; Offer alternatives; Tell predictions; Hunt for discriminating evidence; Evaluate contradictions; Select or revise provisionally; Identify the next evidence need; State limitations and stop boundary.

## Practical application

Prepare a diagnostic hypothesis ledger containing the symptom, sourced observations, system state, competing hypotheses, predictions, discriminating evidence, contradictions, provisional ranking and stop boundary.

## Assessment relevance

A strong response avoids first-guess diagnosis, changes hypothesis ranking when evidence conflicts and does not claim root cause without discriminating evidence.
