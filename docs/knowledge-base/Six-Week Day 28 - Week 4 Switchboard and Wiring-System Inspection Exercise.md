# Six-Week Day 28 - Week 4 Switchboard and Wiring-System Inspection Exercise

## Purpose

Integrate source-state, switchboard-area, route-condition and circuit-role evidence into a bounded paper inspection report while keeping observations, inferences, dependencies and technical authority distinct.

## Core workflow

**I-N-S-P-E-C-T:** Identify scope and sources → Note observations only → Segment the installation → Pair evidence and dependencies → Examine interactions → Classify the claim → Tell the bounded conclusion.

## Evidence and claim model

- Evidence grades: **recalled**, **located**, **supported**, **transferred**, **unresolved**.
- Claim grades: **memory claim**, **provisional interpretation**, **supported study conclusion**, **authorised technical determination**.
- Integrated ledger: item → scope → observation → inference → workflow → evidence → grade → dependencies → interactions → gaps → reopening triggers → claim grade → bounded next action.
- Photographs, labels and memories are evidence inputs, not complete compliance or safety proofs.

## Practical application

Create a source-state map, functional-area map, route ledger, circuit-role table and integrated inspection ledger for a fictional pack. Repeat affected entries after changed source, board-function, route, drawing and identification evidence.

## Assessment relevance

Cumulative Week 4 checkpoint testing observation/inference separation, source-state awareness, segmentation, dependency control, interaction analysis, changed-pack transfer and bounded reporting. The 12-point rubric and critical-error gates are original educational tools, not an official RTO pass mark.

## Reopening triggers

Reopen findings when a source, operating state, drawing, schedule, board function, circuit destination, route condition, identification record, interconnection or inspection scope changes or conflicts with later evidence.
