## Core concept

Day 27 is a sustained, original paper-based mock that integrates navigation, design, installation, verification, interpretation and fault-analysis reasoning. It is not a reproduction of an RTO examination and does not introduce official test procedures, values or acceptance criteria.

The **A-T-T-E-M-P-T** workflow is:

1. **Audit the paper**.
2. **Time-box the sections**.
3. **Tag the task type**.
4. **Extract facts and assumptions**.
5. **Map the reasoning**.
6. **Produce a bounded answer**.
7. **Triage and review**.

## Practical application

The mock contains four original sections:

- rule navigation and concepts;
- design and installation reasoning;
- verification and result interpretation;
- fault analysis and handover.

The learner submits an evidence pack containing assumptions, source-check markers, contradictions, safety stops, confidence and time use. Exact technical requirements must be verified using current authorised sources.

## Assessment relevance

The mock tests integrated performance rather than isolated recall. Review should separate result accuracy, reasoning process, safety, source discipline, communication, confidence and time. High-confidence safety errors receive priority during Day 28.

## Safety and review boundary

Day 27 is `review-required`, safety-critical, `reference_check_required` and not `technically-reviewed`. The module authorises no approach to, operation of, opening of, isolation of, energisation of, testing of, resetting of, repair of or alteration to real electrical equipment.

Qualified review is required before technical marking guidance or an answer key is treated as authoritative.
