## Core workflow

Use **M-O-T-O-R**:

- **Machine and mission**
- **Operating cases**
- **Thermal and fault protection**
- **Operating controls and isolation**
- **Records and review**

A motor installation is a coordinated electromechanical system. The motor, driven machine, cable, starter, contactor, protective functions, control logic, restart behaviour and isolation boundary must be reviewed together.

## Evidence model

Evidence is graded **observed**, **documented**, **manufacturer-verified**, **derived**, **assumed** or **missing**. Claims are graded **described**, **supported**, **verified** or **unresolved**. A change to duty, starting method, supply, control logic or driven load reopens every dependent conclusion.

## Practical application

Build an operating-case register, function map, coordination model and evidence ledger for a fictional motor system. Separate starting, running, overload, short circuit, stall, abnormal supply, remote control, automatic restart and maintenance states. Finish with a bounded claim rather than a protection setting or field procedure.

## Assessment relevance

A strong response distinguishes protection functions, includes the driven machine, checks coordination, uses evidence and claim grades, and recognises that a stationary motor is neither necessarily isolated nor guaranteed to remain stopped.

The module uses a six-category, 12-point educational rubric. A score of 10/12 with no critical error indicates readiness for Day 20C; it is not an official assessment rule.

## Safety boundary

This note authorises no opening, touching, switching, isolation, testing, adjustment, resetting, commissioning, installation or alteration. It provides no settings method, isolation sequence or commissioning procedure.
