# Six-Week Day 37 - Mandatory Test Purposes and Safe Test Preconditions

## Purpose

Match a verification question to its evidence purpose, grade the available evidence and identify every precondition that must be resolved before an authorised method may be selected.

## Core workflow

**P-R-E-C-H-E-C-K:** Purpose → Reference → Establish boundary and sources → Confirm authority → Hazards → Equipment evidence → Circuit state and dependencies → Keep the stop decision.

## Evidence and claim model

- Evidence grades: **stated**, **indicated**, **corroborated**, **transferred**, **unresolved**.
- Claim grades: **assumption**, **provisional educational conclusion**, **supported educational conclusion**, **authorised technical determination**.
- Test-precondition ledger: evidence question → category → boundary/exclusions → sources/interfaces → conceptual state → authorised reference → authority → hazards/controls → instrument evidence → grades → dependencies → reopening triggers → decision.
- A test name, instrument status label, drawing, schedule or previous record may contribute evidence but does not independently establish a complete precondition set or technical conclusion.

## Practical application

Build eight fictional test-precondition ledgers, fade the worked support, then introduce changed-source, changed-document and changed-instrument-evidence conditions. Reopen only affected rows and retain a stop or escalation decision wherever authority, method, state, boundary or equipment evidence is unresolved.

## Assessment relevance

Tests observable performance in purpose definition, boundary and source mapping, conceptual-state reasoning, authority and equipment evidence, evidence grading, stop discipline and selective reopening. The six-category 12-point rubric and critical-error gates are original educational tools, not an official RTO pass mark.

## Reopening triggers

Reopen affected ledger rows when scope, circuit identity, equipment, exclusions, normal or alternate sources, stored energy, automatic restoration, document revision, approved method, acceptance source, instrument status, environment, competence, supervision or authority changes or conflicts with later evidence.
