# Six-Week Day 26 - Rest Visual-Recall Practice and Catch-Up

## Purpose

Use a bounded 30-minute recovery block to reconstruct switchboard and route relationships from memory, grade the supporting evidence, repair no more than three high-value errors and make a readiness decision without adding new electrical theory.

## Core workflow

**R-E-D-R-A-W:** Rate readiness → Evoke the model closed-note → Detect the highest-value gap → Review the smallest source needed → Apply the correction to a changed prompt → Write the readiness decision and stop.

## Evidence and claim model

- Evidence grades: **recalled**, **located**, **supported**, **transferred**, **unresolved**.
- Claim grades: **memory claim**, **provisional correction**, **supported study conclusion**, **authorised technical determination**.
- Error-repair ledger: prompt and relationship → confidence → evidence grade → error type → smallest source checked → bounded correction → changed prompt → transfer result → reopening trigger.
- A fluent or neat redraw does not prove that the remembered relationships are complete or supported.

## Practical application

Reconstruct one switchboard relationship map and one segmented wiring route without notes. Compare only after both drawings are complete, repair up to three high-value errors, then apply one correction to a changed fictional prompt. Stop at 30 minutes or any earlier fatigue, symptom, uncertainty or safety boundary.

## Assessment relevance

Strengthens closed-note relationship recall, misconception detection, evidence calibration, changed-prompt transfer and disciplined readiness decisions. The six-category 12-point rubric and critical-error gates are original educational tools, not an official RTO pass mark.

## Readiness boundary

Choose exactly one result:

- **Ready** — key relationships retrieved, selected corrections supported and transfer completed safely.
- **Limited catch-up required** — one bounded prerequisite gap remains and exactly one next action is recorded.
- **Rest required** — symptoms, fatigue, repeated unsafe claims, unresolved high-confidence errors or scope-control failure make further study unreliable.

Readiness is not technical competence, authorisation or proof of practical skill.
