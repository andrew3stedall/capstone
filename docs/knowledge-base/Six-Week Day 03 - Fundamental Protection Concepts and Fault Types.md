# Six-Week Day 03 — Fundamental Protection Concepts and Fault Types

## Purpose

This note connects safety-pathway analysis to later protection selection. The learner classifies the abnormal condition from facts and current paths before naming a protective device.

## Core model

Use **F-A-U-L-T**:

- **Fix the facts** — record the stated circuit arrangement and symptoms without adding assumptions.
- **Analyse the path** — trace intended current flow and any abnormal path or discontinuity.
- **Understand the mechanism** — distinguish overload, short circuit, earth fault, leakage or residual-current condition, open circuit and unresolved cases.
- **Link the protection purpose** — state whether the immediate concern is heating, shock exposure, fault energy, equipment damage or another consequence.
- **Test the evidence boundary** — identify the authorised source and technical data needed before selection, testing or practical action.

The reusable reasoning chain is:

**condition → current path → mechanism → consequence → protection purpose → evidence boundary**

## Practical application

Classify fictional cards containing:

1. increased demand in an otherwise intact circuit;
2. unintended connection between different potentials;
3. an unintended earth-associated path;
4. current imbalance with unknown cause;
5. discontinuity in an intended path;
6. incomplete evidence that requires provisional classification.

For each, state the known facts, intended path, abnormal path or discontinuity, fault category, possible consequence, protection purpose, missing evidence and stop boundary.

## Assessment relevance

This method supports scenario explanation, protection reasoning, later calculations and fault-finding discipline. It does not provide official assessment criteria, device-selection values or practical procedures.
