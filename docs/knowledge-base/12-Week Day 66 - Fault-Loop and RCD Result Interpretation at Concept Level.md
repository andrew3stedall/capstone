# 12-Week Day 66 - Fault-Loop and RCD Result Interpretation at Concept Level

## Purpose

A concept-level method for linking existing fault-loop and RCD records to the exact protective question, circuit, device, source state and evidence provenance they can support.

## Core model

Use **P-R-O-T-E-C-T**: Pin down the protection question; Record circuit, device and source; Observe the result; Trace dependencies; Examine provenance and contradictions; Constrain interpretation; Trigger review.

## Practical application

Prepare a protective-evidence dependency map that identifies the protection question, circuit, device, supply state, provenance, contradictions, supported claim, unsupported claim and reopening trigger.

## Assessment relevance

A strong response does not interpret a generic pass label. It maintains source-state and device identity, distinguishes plausibility from acceptance and limits each conclusion to its evidence purpose.

This note provides no field method, instrument setup, official sequence, test current, timing limit, acceptance value or practical authority. Exact requirements require current authorised sources and qualified technical review.
