# Six-Week Day 33 - Rest Retrieval and Scenario Triage

## Purpose

Use a bounded recovery session to retrieve Week 5 distinctions, grade evidence, prioritise consequential errors, repair no more than three high-value gaps and make a defensible readiness decision without adding new theory.

## Core workflow

**T-R-I-A-G-E:** Test readiness → Retrieve closed-note distinctions → Identify error and evidence grade → Assign consequence and priority → Give one bounded correction and transfer test → End with readiness and one next action.

## Evidence and claim model

- Evidence grades: **recalled**, **located**, **supported**, **transferred**, **unresolved**.
- Claim grades: **memory claim**, **provisional correction**, **supported study conclusion**, **authorised technical determination**.
- Scenario-triage ledger: prompt → first response → confidence → evidence grade → error type → consequence → priority → smallest source → bounded correction → changed condition → transfer → reopening trigger.
- Confidence, familiarity and speed are not evidence.

## Practical application

Complete six one-minute fictional scenario prompts, grade every response, select no more than three consequential repairs, check only the smallest relevant source, apply one changed condition and record ready, conditional-ready or stop within 30 minutes.

## Assessment relevance

Builds observable performance in closed-note retrieval, category separation, evidence grading, prioritisation, bounded remediation, changed-condition transfer and recovery discipline. The six-category 12-point rubric and critical-error gates are original educational tools, not an official RTO pass mark.

## Reopening triggers

Reopen a correction when the task boundary, location classification, source arrangement, appliance information, motor duty, operating state, control mode, alternate supply, stored energy, reference source, jurisdiction or evidence currency changes or conflicts with later information.
