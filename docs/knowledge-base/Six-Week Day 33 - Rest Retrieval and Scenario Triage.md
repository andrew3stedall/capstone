# Six-Week Day 33 - Rest Retrieval and Scenario Triage
## Purpose
Use a bounded recovery session to retrieve Week 5 distinctions, prioritise consequential errors and make a readiness decision without adding new theory.
## Core workflow
**T-R-I-A-G-E:** Time-box → Retrieve → Identify error → Assign priority → Give one correction → End with readiness.

## Practical application
Classify six short scenarios, repair no more than three high-value errors and record ready, conditional-ready or stop.
## Assessment relevance
Builds fast classification, evidence discipline and remediation selection under a strict time limit.
