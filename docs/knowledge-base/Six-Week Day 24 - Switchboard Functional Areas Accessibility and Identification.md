# Six-Week Day 24 - Switchboard Functional Areas Accessibility and Identification

## Purpose

Map switchboard functions, source relationships, access conditions and identification evidence without judging technical status from appearance alone.

## Core workflow

**B-O-A-R-D-S:** Bound the board and every source → Organise functional areas → Assess access and barriers → Relate identification to evidence → Detect constraints and interactions → State bounded findings.

## Evidence and claim model

- Evidence grades: **supplied**, **corroborated**, **derived**, **assumed**, **missing/conflicting**.
- Claim grades: **observation**, **provisional classification**, **supported paper conclusion**, **authorised technical determination**.
- Board-evidence ledger: feature → function → source relationship → access condition → identification → correspondence → interaction risk → evidence grade → claim grade → dependency → reopening trigger → unresolved check.
- Appearance, labels and physical reach cannot substitute for current correspondence evidence, source mapping or permission.

## Practical application

Construct a board-evidence ledger for a fictional evidence pack, grade every item, identify unresolved source, access, barrier and correspondence conditions, write a bounded finding, then rebuild affected rows after identification, access or task changes.

## Assessment relevance

Tests functional mapping, source awareness, access and permission distinctions, identification correspondence, evidence calibration, changed-condition transfer and bounded reporting. The 12-point rubric and critical-error gates are original educational readiness tools, not an official RTO pass mark.
