# Six-Week Day 21 - Week 3 Integrated Circuit-Design Exercise

## Purpose
Integrate the Week 3 design chain into a timed, evidence-controlled response without treating one plausible calculation or conductor candidate as a complete design proof.

## Core workflow
**I-N-T-E-G-R-A-T-E:** Interpret → Name evidence → Test cases → Execute → Govern → Reopen → Audit → Tell → Evaluate.

## Evidence and claim model
- Evidence grades: **supplied**, **corroborated**, **derived**, **assumed**, **missing/conflicting**.
- Claim grades: **observation**, **provisional reasoning**, **supported paper conclusion**, **authorised technical determination**.
- Integration ledger: gate or claim → evidence grade → dependencies → present result → unresolved evidence → reopening trigger.
- A passing result at one gate cannot compensate for an unresolved critical dependency elsewhere in the chain.

Complete a fictional brief, operating-case comparison, evidence-grade ledger, full design-gate record, governing-condition explanation, changed-condition reopening map and bounded recommendation.

## Assessment relevance
