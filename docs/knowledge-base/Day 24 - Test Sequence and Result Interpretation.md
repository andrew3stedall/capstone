## Core concept

A verification sequence is a dependency-controlled evidence process. Each stage must have established prerequisites, a defined purpose, an authorised method and criterion, a stop gate and a bounded relationship to later conclusions.

The **O-R-D-E-R** workflow is:

1. **Outline the scope and sources**.
2. **Retrieve the authorised sequence**.
3. **Define dependencies and stop gates**.
4. **Evaluate each result in context**.
5. **Record a bounded conclusion**.

A result is not self-interpreting. Subject, installation state, approved method, instrument, units, criterion, limitations and corroborating evidence determine what it can support.

## Practical application

Use a dependency map to separate:

- prerequisite, dependent, corroborating, blocking and confirmation evidence;
- the installation and source state for each stage;
- equipment and manufacturer constraints;
- progression gates and stop conditions;
- the exact later decision each result may support.

Incomplete or contradictory records should be classified as not demonstrated, inconclusive or requiring resolution rather than converted into assumed pass results.

## Assessment relevance

This block prepares the learner to explain why test order matters, detect invalid progression, interpret evidence within scope and identify when a favourable-looking result remains insufficient. It provides the evidence-discipline foundation for Day 25 systematic fault finding.

## Safety and review boundary

This note does not provide an official test order, instrument setup, connection method, test parameter, acceptance value, energisation instruction, live-testing process or certification procedure. Current authorised sources, approved RTO or workplace procedures, manufacturer information and qualified technical review are required.
