# Six-Week Day 27 - Consumer Mains Submains and Final-Subcircuit Roles

## Purpose

Classify circuit-role candidates from evidenced boundaries, sources, destinations and distribution function; preserve unresolved results; and reopen dependent decisions when the architecture changes.

## Core workflow

**B-O-U-N-D-S:** Bound the segment → Observe every source → Understand the destination → Name the supplied function → Distinguish evidence from appearance → State the bounded classification, dependencies and reopening triggers.

## Evidence and claim model

- Evidence grades: **recalled**, **located**, **supported**, **transferred**, **unresolved**.
- Claim grades: **memory claim**, **provisional classification**, **supported study conclusion**, **authorised technical determination**.
- Circuit-role evidence ledger: segment → boundaries → source/state → destination function → evidence → grade → proposed role → dependencies → reopening triggers → downstream checks.
- Cable size, route, enclosure type, label and location are clues only; none independently proves circuit role.

## Practical application

Complete the circuit-role evidence ledger for fictional segments, then repeat it after changed board function, alternate-source, destination and source-record conditions. Use unresolved rather than inventing a boundary.

## Assessment relevance

Tests architecture tracing, terminology, evidence control, source-state awareness, dependency reasoning, changed-condition transfer and the ability to separate an educational candidate from an authorised technical determination. The 12-point rubric and critical-error gates are original educational tools, not an official RTO pass mark.

## Reopening triggers

Reopen classification and downstream decisions when a supply boundary, source, operating state, destination, distribution function, board arrangement, drawing, schedule or interconnection changes or conflicts with later evidence.
