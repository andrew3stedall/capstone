## Core concepts

- **Interpretation:** explaining a result within its stated boundary and evidence.
- **Acceptance criterion:** an authorised requirement used to judge a result.
- **Design iteration:** a controlled change followed by recalculation and reopened checks.
- **Sensitivity:** how strongly a result responds to an input change.
- **Reopening trigger:** a changed fact that weakens an earlier conclusion.
- **Claim grade:** described, supported or verified.

## I-T-E-R-A-T-E workflow

Identify the complete boundary; trace the criterion; evaluate the current result; revise one variable; audit reopened gates; test alternatives consistently; explain the bounded recommendation.

## Practical application

Learners create an interpretation ledger, predict sensitivity, recalculate one changed input, compare three candidates and withdraw conclusions invalidated by a changed operating condition.

## Assessment relevance

Observable evidence includes boundary completeness, criterion provenance, like-for-like comparison, reopening discipline, trade-off explanation and restraint in technical claims.
